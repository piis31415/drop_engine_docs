Type\+Script is more syntactically {\bfseries{flexible}} than JSON. Comments are allowed. Keys needn\textquotesingle{}t be quoted. Arrays can have trailing commas.

Type\+Script allows a little {\bfseries{programming}} in config files. Share variables, use utility functions, etc.

Type\+Script {\bfseries{types}} provide free error checking, and free IDE support for getting config files right.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2a393ff5ed6126044591ec39758988fc0_autotoc_md3531}{}\doxysubsection{\texorpdfstring{Fast}{Fast}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2a393ff5ed6126044591ec39758988fc0_autotoc_md3531}
Parsing Type\+Script config files is plenty quick. config-\/file-\/ts caches the Type\+Script output.
