{\bfseries{node-\/addon-\/api}} provides a \href{../tools/check-napi.js}{\texttt{ checker tool}} that will inspect a given directory tree, identifying all Node.\+js native addons therein, and further indicating for each addon whether it is an N-\/\+API addon.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2a8d746533630015e8966d6d90b186401_autotoc_md7744}{}\doxysection{\texorpdfstring{To use the checker tool\+:}{To use the checker tool\+:}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2a8d746533630015e8966d6d90b186401_autotoc_md7744}

\begin{DoxyEnumerate}
\item Install the application with {\ttfamily npm install}.
\item If the application does not depend on {\bfseries{node-\/addon-\/api}}, copy the checker tool into the application\textquotesingle{}s directory.
\item If the application does not depend on {\bfseries{node-\/addon-\/api}}, run the checker tool from the application\textquotesingle{}s directory\+:

{\ttfamily sh node ./check-\/napi.js }

Otherwise, the checker tool can be run from the application\textquotesingle{}s {\ttfamily node\+\_\+modules/} subdirectory\+:

{\ttfamily sh node ./node\+\_\+modules/node-\/addon-\/api/tools/check-\/napi.js }
\end{DoxyEnumerate}

The tool accepts the root directory from which to start checking for Node.\+js native addons as a single optional command line parameter. If ommitted it will start checking from the current directory ({\ttfamily .}).
