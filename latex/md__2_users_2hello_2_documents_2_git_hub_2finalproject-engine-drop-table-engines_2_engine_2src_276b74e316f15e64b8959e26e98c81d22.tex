The expansion engine roughly has the following rules\+:


\begin{DoxyItemize}
\item {\ttfamily \$KEY} will expand any env with the name {\ttfamily KEY}
\item {\ttfamily \$\{KEY\}} will expand any env with the name {\ttfamily KEY}
\item {\ttfamily \textbackslash{}\$KEY} will escape the {\ttfamily \$KEY} rather than expand
\item {\ttfamily \$\{KEY\+:-\/default\}} will first attempt to expand any env with the name {\ttfamily KEY}. If not one, then it will return {\ttfamily default}
\item {\ttfamily \$\{KEY-\/default\}} will first attempt to expand any env with the name {\ttfamily KEY}. If not one, then it will return {\ttfamily default}
\end{DoxyItemize}

You can see a full list of rules \href{https://dotenvx.com/docs/env-file\#interpolation}{\texttt{ here}}.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_276b74e316f15e64b8959e26e98c81d22_autotoc_md3939}{}\doxysubsubsection{\texorpdfstring{How can I avoid expanding pre-\/existing envs (already in my {\ttfamily process.\+env}, for example {\ttfamily pas\$word})?}{How can I avoid expanding pre-\/existing envs (already in my {\ttfamily process.\+env}, for example {\ttfamily pas\$word})?}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_276b74e316f15e64b8959e26e98c81d22_autotoc_md3939}
