{\ttfamily \doxylink{class_napi_1_1_function_reference}{Napi\+::\+Function\+Reference}} is a subclass of \doxysectlink{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fe3873f06559c5a0ab6a1717ffe1ef05}{{\ttfamily Napi\+::Reference}}{0}, and is equivalent to an instance of {\ttfamily \doxylink{class_napi_1_1_reference}{Napi\+::\+Reference}\texorpdfstring{$<$}{<}\doxylink{class_napi_1_1_function}{Napi\+::\+Function}\texorpdfstring{$>$}{>}}. This means that a {\ttfamily \doxylink{class_napi_1_1_function_reference}{Napi\+::\+Function\+Reference}} holds a \doxysectlink{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_26fc3837f8c0f1284160c3afbc00a86e7}{{\ttfamily Napi\+::Function}}{0}, and a count of the number of references to that {\ttfamily \doxylink{class_napi_1_1_function}{Napi\+::\+Function}}. When the count is greater than 0, a {\ttfamily \doxylink{class_napi_1_1_function_reference}{Napi\+::\+Function\+Reference}} is not eligible for garbage collection. This ensures that the {\ttfamily Function} will remain accessible, even if the original reference to it is no longer available. {\ttfamily \doxylink{class_napi_1_1_function_reference}{Napi\+::\+Function\+Reference}} allows the referenced Java\+Script function object to be called from a native add-\/on with two different methods\+: {\ttfamily Call} and {\ttfamily Make\+Callback}. See the documentation for \doxysectlink{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_26fc3837f8c0f1284160c3afbc00a86e7}{{\ttfamily Napi\+::Function}}{0} for when {\ttfamily Call} should be used instead of {\ttfamily Make\+Callback} and vice-\/versa.
