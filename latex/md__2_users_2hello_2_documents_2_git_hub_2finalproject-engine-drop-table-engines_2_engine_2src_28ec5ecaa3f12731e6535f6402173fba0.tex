For those of you wondering about the difference between these \mbox{[}popular\mbox{]} NPM modules, here\textquotesingle{}s a brief rundown ...

\href{https://github.com/jshttp/mime-db}{\texttt{ {\ttfamily mime-\/db}}} is "{}the source of truth"{} for MIME type information. It is not an API. Rather, it is a canonical dataset of mime type definitions pulled from IANA, Apache, NGINX, and custom mappings submitted by the Node.\+js community.

\href{https://github.com/jshttp/mime-types}{\texttt{ {\ttfamily mime-\/types}}} is a thin wrapper around mime-\/db that provides an API drop-\/in compatible(ish) with {\ttfamily mime @ \texorpdfstring{$<$}{<} v1.\+3.\+6} API.

{\ttfamily mime} is, as of v2, a self-\/contained module bundled with a pre-\/optimized version of the {\ttfamily mime-\/db} dataset. It provides a simplified API with the following characteristics\+:


\begin{DoxyItemize}
\item Intelligently resolved type conflicts (See \href{https://github.com/broofa/mime-score}{\texttt{ mime-\/score}} for details)
\item Method naming consistent with industry best-\/practices
\item Compact footprint. E.\+g. The minified+compressed sizes of the various modules\+:
\end{DoxyItemize}
