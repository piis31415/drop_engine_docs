zip-\/stream is a streaming zip archive generator based on the {\ttfamily Zip\+Archive\+Output\+Stream} prototype found in the \href{https://www.npmjs.org/package/compress-commons}{\texttt{ compress-\/commons}} project.

It was originally created to be a successor to \href{https://npmjs.org/package/zipstream}{\texttt{ zipstream}}.

This module is meant to be wrapped internally by other modules and therefore lacks any queue management. This means you have to wait until the previous entry has been fully consumed to add another. Nested callbacks should be used to add multiple entries. There are modules like \href{https://npmjs.org/package/async}{\texttt{ async}} that ease the so called "{}callback hell"{}.

If you want a module that handles entry queueing and much more, you should check out \href{https://npmjs.org/package/archiver}{\texttt{ archiver}} which uses this module internally.
