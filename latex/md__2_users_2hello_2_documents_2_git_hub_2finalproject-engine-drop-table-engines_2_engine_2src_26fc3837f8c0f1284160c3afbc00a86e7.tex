The {\ttfamily \doxylink{class_napi_1_1_function}{Napi\+::\+Function}} class provides a set of methods for creating a function object in native code that can later be called from Java\+Script. The created function is not automatically visible from Java\+Script. Instead it needs to be part of the add-\/on\textquotesingle{}s module exports or be returned by one of the module\textquotesingle{}s exported functions.

In addition the {\ttfamily \doxylink{class_napi_1_1_function}{Napi\+::\+Function}} class also provides methods that can be used to call functions that were created in Java\+Script and passed to the native add-\/on.

The {\ttfamily \doxylink{class_napi_1_1_function}{Napi\+::\+Function}} class inherits its behavior from the {\ttfamily \doxylink{class_napi_1_1_object}{Napi\+::\+Object}} class (for more info see\+: \doxysectlink{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2664783051e3556aa75aa8a5d3640a968}{{\ttfamily Napi\+::Object}}{0}).\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_26fc3837f8c0f1284160c3afbc00a86e7_autotoc_md7833}{}\doxysection{\texorpdfstring{Example}{Example}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_26fc3837f8c0f1284160c3afbc00a86e7_autotoc_md7833}

With the {\ttfamily \doxylink{class_napi_1_1_function}{Napi\+::\+Function}} class it is possible to call a Java\+Script function object from a native add-\/on with two different methods\+: {\ttfamily Call} and {\ttfamily Make\+Callback}. The API of these two methods is very similar, but they are used in different contexts. The {\ttfamily Make\+Callback} method is used to call from native code back into Java\+Script after returning from an \doxysectlink{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_29e0fc6d9e899f865bc8516295092586e}{asynchronous operation}{0} and in general in situations which don\textquotesingle{}t have an existing Java\+Script function on the stack. The {\ttfamily Call} method is used when there is already a Java\+Script function on the stack (for example when running a native method called from Java\+Script).\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_26fc3837f8c0f1284160c3afbc00a86e7_autotoc_md7834}{}\doxysection{\texorpdfstring{Methods}{Methods}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_26fc3837f8c0f1284160c3afbc00a86e7_autotoc_md7834}
