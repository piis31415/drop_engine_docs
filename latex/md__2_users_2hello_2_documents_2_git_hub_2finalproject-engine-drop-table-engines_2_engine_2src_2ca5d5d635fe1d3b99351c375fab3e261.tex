A fork of \href{https://github.com/sindresorhus/move-file}{\texttt{ move-\/file}} with compatibility with all node 10.\+x versions.

\begin{quote}
Move a file (or directory) \end{quote}
The built-\/in \href{https://nodejs.org/api/fs.html\#fs_fs_rename_oldpath_newpath_callback}{\texttt{ {\ttfamily fs.\+rename()}}} is just a Java\+Script wrapper for the C {\ttfamily rename(2)} function, which doesn\textquotesingle{}t support moving files across partitions or devices. This module is what you would have expected {\ttfamily fs.\+rename()} to be.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2ca5d5d635fe1d3b99351c375fab3e261_autotoc_md1272}{}\doxysection{\texorpdfstring{Highlights}{Highlights}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2ca5d5d635fe1d3b99351c375fab3e261_autotoc_md1272}

\begin{DoxyItemize}
\item Promise API.
\item Supports moving a file across partitions and devices.
\item Optionally prevent overwriting an existing file.
\item Creates non-\/existent destination directories for you.
\item Support for Node versions that lack built-\/in recursive {\ttfamily fs.\+mkdir()}
\item Automatically recurses when source is a directory.
