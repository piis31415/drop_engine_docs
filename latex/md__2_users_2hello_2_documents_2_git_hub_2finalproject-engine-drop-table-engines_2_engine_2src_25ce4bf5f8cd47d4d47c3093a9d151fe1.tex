EJS ships with a full-\/featured CLI. Options are similar to those used in Java\+Script code\+:


\begin{DoxyItemize}
\item {\ttfamily -\/o / -\/-\/output-\/file FILE} Write the rendered output to FILE rather than stdout.
\item {\ttfamily -\/f / -\/-\/data-\/file FILE} Must be JSON-\/formatted. Use parsed input from FILE as data for rendering.
\item {\ttfamily -\/i / -\/-\/data-\/input STRING} Must be JSON-\/formatted and URI-\/encoded. Use parsed input from STRING as data for rendering.
\item {\ttfamily -\/m / -\/-\/delimiter CHARACTER} Use CHARACTER with angle brackets for open/close (defaults to \%).
\item {\ttfamily -\/p / -\/-\/open-\/delimiter CHARACTER} Use CHARACTER instead of left angle bracket to open.
\item {\ttfamily -\/c / -\/-\/close-\/delimiter CHARACTER} Use CHARACTER instead of right angle bracket to close.
\item {\ttfamily -\/s / -\/-\/strict} When set to {\ttfamily true}, generated function is in strict mode
\item {\ttfamily -\/n / -\/-\/no-\/with} Use \textquotesingle{}locals\textquotesingle{} object for vars rather than using {\ttfamily with} (implies --strict).
\item {\ttfamily -\/l / -\/-\/locals-\/name} Name to use for the object storing local variables when not using {\ttfamily with}.
