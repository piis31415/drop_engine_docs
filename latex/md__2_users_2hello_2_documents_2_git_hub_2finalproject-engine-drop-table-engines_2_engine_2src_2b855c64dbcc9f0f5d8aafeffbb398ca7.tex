Some core ESLint rules do not support Type\+Script syntax\+: either they crash, ignore the syntax, or falsely report against it. In these cases, we create what we call an "{}extension rule"{}\+: a rule within our plugin that has the same functionality, but also supports Type\+Script.

Extension rules generally completely replace the base rule from ESLint core. If the base rule is enabled in a config you extend from, you\textquotesingle{}ll need to disable the base rule\+:
