Syntax sugar for the combination of minimum and maximum keywords, also fails schema compilation if there are no numbers in the range.

The value of this keyword must be the array consisting of two numbers, the second must be greater or equal than the first one.

If the validated value is not a number the validation passes, otherwise to pass validation the value should be greater (or equal) than the first number and smaller (or equal) than the second number in the array. If {\ttfamily exclusive\+Range} keyword is present in the same schema and its value is true, the validated value must not be equal to the range boundaries.

These keywords allow to define minimum/maximum constraints when the format keyword defines ordering.

These keywords apply only to strings. If the data is not a string, the validation succeeds.

The value of keyword {\ttfamily format\+Maximum} ({\ttfamily format\+Minimum}) should be a string. This value is the maximum (minimum) allowed value for the data to be valid as determined by {\ttfamily format} keyword. If {\ttfamily format} is not present schema compilation will throw exception.

When this keyword is added, it defines comparison rules for formats {\ttfamily "{}date"{}}, {\ttfamily "{}time"{}} and {\ttfamily "{}date-\/time"{}}. Custom formats also can have comparison rules. See \href{https://github.com/epoberezkin/ajv\#api-addformat}{\texttt{ add\+Format}} method.

The value of keyword {\ttfamily format\+Exclusive\+Maximum} ({\ttfamily format\+Exclusive\+Minimum}) should be a boolean value. These keyword cannot be used without {\ttfamily format\+Maximum} ({\ttfamily format\+Minimum}). If this keyword value is equal to {\ttfamily true}, the data to be valid should not be equal to the value in {\ttfamily format\+Maximum} ({\ttfamily format\+Minimum}) keyword.

This keyword allows a string to be modified before validation.

These keywords apply only to strings. If the data is not a string, the transform is skipped.

There are limitation due to how ajv is written\+:
\begin{DoxyItemize}
\item a stand alone string cannot be transformed. ie `data = \textquotesingle{}a'; ajv.\+validate(schema, data);{\ttfamily }
\item {\ttfamily currently cannot work with}ajv-\/pack\`{}
\end{DoxyItemize}

{\bfseries{Supported options\+:}}
\begin{DoxyItemize}
\item {\ttfamily trim}\+: remove whitespace from start and end
\item {\ttfamily trim\+Left}\+: remove whitespace from start
\item {\ttfamily trim\+Right}\+: remove whitespace from end
\item {\ttfamily to\+Lower\+Case}\+: case string to all lower case
\item {\ttfamily to\+Upper\+Case}\+: case string to all upper case
\item {\ttfamily to\+Enum\+Case}\+: case string to match case in schema
\end{DoxyItemize}

Options are applied in the order they are listed.

Note\+: {\ttfamily to\+Enum\+Case} requires that all allowed values are unique when case insensitive.

This keyword allows to require the presence of all properties used in {\ttfamily properties} keyword in the same schema object.

This keyword applies only to objects. If the data is not an object, the validation succeeds.

The value of this keyword must be boolean.

If the value of the keyword is {\ttfamily false}, the validation succeeds.

If the value of the keyword is {\ttfamily true}, the validation succeeds if the data contains all properties defined in {\ttfamily properties} keyword (in the same schema object).

If the {\ttfamily properties} keyword is not present in the same schema object, schema compilation will throw exception.

These keywords allow to choose the schema to validate the data based on the value of some property in the validated data.

These keywords must be present in the same schema object ({\ttfamily select\+Default} is optional).

The value of {\ttfamily select} keyword should be a \href{https://github.com/epoberezkin/ajv/tree/5.0.2-beta.0\#data-reference}{\texttt{ \$data reference}} that points to any primitive JSON type (string, number, boolean or null) in the data that is validated. You can also use a constant of primitive type as the value of this keyword (e.\+g., for debugging purposes).

The value of {\ttfamily select\+Cases} keyword must be an object where each property name is a possible string representation of the value of {\ttfamily select} keyword and each property value is a corresponding schema (from draft-\/06 it can be boolean) that must be used to validate the data.

The value of {\ttfamily select\+Default} keyword is a schema (from draft-\/06 it can be boolean) that must be used to validate the data in case {\ttfamily select\+Cases} has no key equal to the stringified value of {\ttfamily select} keyword.

The validation succeeds in one of the following cases\+:
\begin{DoxyItemize}
\item the validation of data using selected schema succeeds,
\item none of the schemas is selected for validation,
\item the value of select is undefined (no property in the data that the data reference points to).
\end{DoxyItemize}

If {\ttfamily select} value (in data) is not a primitive type the validation fails.

{\bfseries{Please note}}\+: these keywords require Ajv {\ttfamily \$data} option to support \href{https://github.com/epoberezkin/ajv/tree/5.0.2-beta.0\#data-reference}{\texttt{ \$data reference}}.
