Reuse your objects and functions for maximum speed. This technique will make any function run \texorpdfstring{$\sim$}{\string~}10\% faster. You call your functions a lot, and it adds up quickly in hot code paths.

Also note how in the above examples, the code, that consumes an istance of {\ttfamily My\+Object}, reset the state to initial condition, just before storing it in the cache. That\textquotesingle{}s needed so that every subsequent request for an instance from the cache, could get a clean instance.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_212219e8579f7397039382fa038e11562_autotoc_md9173}{}\doxysection{\texorpdfstring{Why}{Why}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_212219e8579f7397039382fa038e11562_autotoc_md9173}
It is faster because V8 doesn\textquotesingle{}t have to collect all the functions you create. On a short-\/lived benchmark, it is as fast as creating the nested function, but on a longer time frame it creates less pressure on the garbage collector.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_212219e8579f7397039382fa038e11562_autotoc_md9174}{}\doxysection{\texorpdfstring{Other examples}{Other examples}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_212219e8579f7397039382fa038e11562_autotoc_md9174}
