The module for calculating Cyclic Redundancy Check (CRC) for Node.\+js and the Browser.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2ae24c68423132ca24836fa34728d0d48_autotoc_md3589}{}\doxysection{\texorpdfstring{Features}{Features}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2ae24c68423132ca24836fa34728d0d48_autotoc_md3589}

\begin{DoxyItemize}
\item Full test suite comparing values against reference {\ttfamily pycrc} implementation.
\item Pure Java\+Script implementation, no dependencies.
\item ES6 modules.
\item Provides CRC tables for optimized calculations.
\item Provides support for the following CRC algorithms\+:
\begin{DoxyItemize}
\item CRC1 ({\ttfamily crc1})
\item CRC8 ({\ttfamily crc8})
\item CRC8 1-\/Wire ({\ttfamily crc81wire})
\item CRC16 ({\ttfamily crc16})
\item CRC16 CCITT ({\ttfamily crc16ccitt})
\item CRC16 Modbus ({\ttfamily crc16modbus})
\item CRC16 Kermit ({\ttfamily crc16kermit})
\item CRC16 XModem ({\ttfamily crc16xmodem})
\item CRC24 ({\ttfamily crc24})
\item CRC32 ({\ttfamily crc32})
\item CRCJAM ({\ttfamily crcjam})
\end{DoxyItemize}
\end{DoxyItemize}\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2ae24c68423132ca24836fa34728d0d48_autotoc_md3590}{}\doxysection{\texorpdfstring{Installation}{Installation}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2ae24c68423132ca24836fa34728d0d48_autotoc_md3590}

Calculate a CRC32 (recommended way)\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ crc32\ from\ 'crc/crc32';}
\DoxyCodeLine{crc32('hello').toString(16);}
\DoxyCodeLine{//\ "{}3610a686"{}}

\end{DoxyCode}


Import everything (this may bloat bundle size)\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ crc\ from\ 'crc';}
\DoxyCodeLine{crc.crc32('hello').toString(16);}
\DoxyCodeLine{//\ "{}3610a686"{}}

\end{DoxyCode}


Or use Common\+JS (compatability mode, no longer recommended)\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ \{\ crc32\ \}\ =\ require('crc');}
\DoxyCodeLine{crc32('hello').toString(16);}
\DoxyCodeLine{//\ "{}3610a686"{}}

\end{DoxyCode}


Calculate a CRC32 of a file\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{crc32(fs.readFileSync('README.md',\ 'utf8')).toString(16);}
\DoxyCodeLine{//\ "{}127ad531"{}}

\end{DoxyCode}


Or using a {\ttfamily Buffer}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{crc32(fs.readFileSync('README.md',\ 'utf8')).toString(16);}
\DoxyCodeLine{//\ "{}127ad531"{}}

\end{DoxyCode}

There\textquotesingle{}s was a bug in Node \href{https://github.com/nodejs/node/issues/9342}{\texttt{ \#9342}} that affected CRC calculation if {\ttfamily Buffer.\+split()} is used (see issue discussion for details). This affected all version starting from {\ttfamily 6.\+3.\+0} up to but not including {\ttfamily 6.\+9.\+2}. The patch \href{https://github.com/nodejs/node/pull/9341}{\texttt{ \#9341}} was released in {\ttfamily 6.\+9.\+2}.
