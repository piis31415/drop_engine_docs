\doxysection*{Rollup}\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fba847dc2b3a96ebc1932ed8018a606c_autotoc_md9289}{}\doxysubsection{\texorpdfstring{Overview}{Overview}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fba847dc2b3a96ebc1932ed8018a606c_autotoc_md9289}
Rollup is a module bundler for Java\+Script which compiles small pieces of code into something larger and more complex, such as a library or application. It uses the standardized ES module format for code, instead of previous idiosyncratic solutions such as Common\+JS and AMD. ES modules let you freely and seamlessly combine the most useful individual functions from your favorite libraries. Rollup can optimize ES modules for faster native loading in modern browsers, or output a legacy module format allowing ES module workflows today.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fba847dc2b3a96ebc1932ed8018a606c_autotoc_md9290}{}\doxysubsection{\texorpdfstring{Quick Start Guide}{Quick Start Guide}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fba847dc2b3a96ebc1932ed8018a606c_autotoc_md9290}
Install with {\ttfamily npm install -\/-\/global rollup}. Rollup can be used either through a \href{https://rollupjs.org/command-line-interface/}{\texttt{ command line interface}} with an optional configuration file or else through its \href{https://rollupjs.org/javascript-api/}{\texttt{ Java\+Script API}}. Run {\ttfamily rollup -\/-\/help} to see the available options and parameters. The starter project templates, \href{https://github.com/rollup/rollup-starter-lib}{\texttt{ rollup-\/starter-\/lib}} and \href{https://github.com/rollup/rollup-starter-app}{\texttt{ rollup-\/starter-\/app}}, demonstrate common configuration options, and more detailed instructions are available throughout the \href{https://rollupjs.org/introduction/}{\texttt{ user guide}}.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fba847dc2b3a96ebc1932ed8018a606c_autotoc_md9291}{}\doxysubsubsection{\texorpdfstring{Commands}{Commands}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fba847dc2b3a96ebc1932ed8018a606c_autotoc_md9291}

Developing software is usually easier if you break your project into smaller separate pieces, since that often removes unexpected interactions and dramatically reduces the complexity of the problems you\textquotesingle{}ll need to solve, and simply writing smaller projects in the first place \href{https://medium.com/@Rich_Harris/small-modules-it-s-not-quite-that-simple-3ca532d65de4}{\texttt{ isn\textquotesingle{}t necessarily the answer}}. Unfortunately, Java\+Script has not historically included this capability as a core feature in the language.

This finally changed with ES modules support in Java\+Script, which provides a syntax for importing and exporting functions and data so they can be shared between separate scripts. Most browsers and Node.\+js support ES modules. However, Node.\+js releases before 12.\+17 support ES modules only behind the {\ttfamily -\/-\/experimental-\/modules} flag, and older browsers like Internet Explorer do not support ES modules at all. Rollup allows you to write your code using ES modules, and run your application even in environments that do not support ES modules natively. For environments that support them, Rollup can output optimized ES modules; for environments that don\textquotesingle{}t, Rollup can compile your code to other formats such as Common\+JS modules, AMD modules, and IIFE-\/style scripts. This means that you get to {\itshape write future-\/proof code}, and you also get the tremendous benefits of...\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fba847dc2b3a96ebc1932ed8018a606c_autotoc_md9293}{}\doxysubsection{\texorpdfstring{Tree Shaking}{Tree Shaking}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fba847dc2b3a96ebc1932ed8018a606c_autotoc_md9293}
In addition to enabling the use of ES modules, Rollup also statically analyzes and optimizes the code you are importing, and will exclude anything that isn\textquotesingle{}t actually used. This allows you to build on top of existing tools and modules without adding extra dependencies or bloating the size of your project.

For example, with Common\+JS, the {\itshape entire tool or library must be imported}.


\begin{DoxyCode}{0}
\DoxyCodeLine{//\ import\ the\ entire\ utils\ object\ with\ CommonJS}
\DoxyCodeLine{var\ utils\ =\ require('node:utils');}
\DoxyCodeLine{var\ query\ =\ 'Rollup';}
\DoxyCodeLine{//\ use\ the\ ajax\ method\ of\ the\ utils\ object}
\DoxyCodeLine{utils.ajax('https://api.example.com?search='\ +\ query).then(handleResponse);}

\end{DoxyCode}


But with ES modules, instead of importing the whole {\ttfamily utils} object, we can just import the one {\ttfamily ajax} function we need\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{//\ import\ the\ ajax\ function\ with\ an\ ES\ import\ statement}
\DoxyCodeLine{import\ \{\ ajax\ \}\ from\ 'node:utils';}
\DoxyCodeLine{}
\DoxyCodeLine{var\ query\ =\ 'Rollup';}
\DoxyCodeLine{//\ call\ the\ ajax\ function}
\DoxyCodeLine{ajax('https://api.example.com?search='\ +\ query).then(handleResponse);}

\end{DoxyCode}


Because Rollup includes the bare minimum, it results in lighter, faster, and less complicated libraries and applications. Since this approach is based on explicit {\ttfamily import} and {\ttfamily export} statements, it is vastly more effective than simply running an automated minifier to detect unused variables in the compiled output code.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fba847dc2b3a96ebc1932ed8018a606c_autotoc_md9294}{}\doxysubsection{\texorpdfstring{Compatibility}{Compatibility}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fba847dc2b3a96ebc1932ed8018a606c_autotoc_md9294}

To make sure your ES modules are immediately usable by tools that work with Common\+JS such as Node.\+js and webpack, you can use Rollup to compile to UMD or Common\+JS format, and then point to that compiled version with the {\ttfamily main} property in your {\ttfamily package.\+json} file. If your {\ttfamily package.\+json} file also has a {\ttfamily module} field, ES-\/module-\/aware tools like Rollup and \href{https://webpack.js.org/}{\texttt{ webpack}} will \href{https://github.com/rollup/rollup/wiki/pkg.module}{\texttt{ import the ES module version}} directly.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fba847dc2b3a96ebc1932ed8018a606c_autotoc_md9297}{}\doxysubsection{\texorpdfstring{Contributors}{Contributors}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fba847dc2b3a96ebc1932ed8018a606c_autotoc_md9297}
