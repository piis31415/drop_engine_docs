To specify another location to download Electron assets from, the following options are available\+:


\begin{DoxyItemize}
\item {\ttfamily mirror\+Options} Object
\begin{DoxyItemize}
\item {\ttfamily mirror} String (optional) -\/ The base URL of the mirror to download from.
\item {\ttfamily nightly\+Mirror} String (optional) -\/ The Electron nightly-\/specific mirror URL.
\item {\ttfamily custom\+Dir} String (optional) -\/ The name of the directory to download from, often scoped by version number.
\item {\ttfamily custom\+Filename} String (optional) -\/ The name of the asset to download.
\item {\ttfamily resolve\+Asset\+URL} Function (optional) -\/ A function allowing customization of the url used to download the asset.
\end{DoxyItemize}
\end{DoxyItemize}

Mirror options can also be specified via the following environment variables\+:
\begin{DoxyItemize}
\item {\ttfamily ELECTRON\+\_\+\+CUSTOM\+\_\+\+DIR} -\/ Specifies the custom directory to download from.
\item {\ttfamily ELECTRON\+\_\+\+CUSTOM\+\_\+\+FILENAME} -\/ Specifies the custom file name to download.
\item {\ttfamily ELECTRON\+\_\+\+MIRROR} -\/ Specifies the URL of the server to download from if the version is not a nightly version.
\item {\ttfamily ELECTRON\+\_\+\+NIGHTLY\+\_\+\+MIRROR} -\/ Specifies the URL of the server to download from if the version is a nightly version.
\end{DoxyItemize}\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_28930e523f2ca815c97eac68de6fcdbd6_autotoc_md251}{}\doxysubsection{\texorpdfstring{Overriding the version downloaded}{Overriding the version downloaded}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_28930e523f2ca815c97eac68de6fcdbd6_autotoc_md251}
The version downloaded can be overriden by setting the {\ttfamily ELECTRON\+\_\+\+CUSTOM\+\_\+\+VERSION} environment variable. Setting this environment variable will override the version passed in to {\ttfamily download} or {\ttfamily download\+Artifact}.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_28930e523f2ca815c97eac68de6fcdbd6_autotoc_md252}{}\doxysection{\texorpdfstring{How It Works}{How It Works}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_28930e523f2ca815c97eac68de6fcdbd6_autotoc_md252}
This module downloads Electron to a known place on your system and caches it so that future requests for that asset can be returned instantly. The cache locations are\+:


\begin{DoxyItemize}
\item Linux\+: {\ttfamily \$XDG\+\_\+\+CACHE\+\_\+\+HOME} or {\ttfamily \texorpdfstring{$\sim$}{\string~}/.cache/electron/}
\item Mac\+OS\+: {\ttfamily \texorpdfstring{$\sim$}{\string~}/\+Library/\+Caches/electron/}
\item Windows\+: {\ttfamily LOCALAPPDATA\%/electron/\+Cache} or {\ttfamily \texorpdfstring{$\sim$}{\string~}/\+App\+Data/\+Local/electron/\+Cache/}
\end{DoxyItemize}

By default, the module uses \href{https://github.com/sindresorhus/got}{\texttt{ {\ttfamily got}}} as the downloader. As a result, you can use the same \href{https://github.com/sindresorhus/got\#options}{\texttt{ options}} via {\ttfamily download\+Options}.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_28930e523f2ca815c97eac68de6fcdbd6_autotoc_md253}{}\doxysubsection{\texorpdfstring{Progress Bar}{Progress Bar}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_28930e523f2ca815c97eac68de6fcdbd6_autotoc_md253}
By default, a progress bar is shown when downloading an artifact for more than 30 seconds. To disable, set the {\ttfamily ELECTRON\+\_\+\+GET\+\_\+\+NO\+\_\+\+PROGRESS} environment variable to any non-\/empty value, or set {\ttfamily quiet} to {\ttfamily true} in {\ttfamily download\+Options}. If you need to monitor progress yourself via the API, set {\ttfamily get\+Progress\+Callback} in {\ttfamily download\+Options}, which has the same function signature as {\ttfamily got}\textquotesingle{}s \href{https://github.com/sindresorhus/got\#ondownloadprogress-progress}{\texttt{ {\ttfamily download\+Progress} event callback}}.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_28930e523f2ca815c97eac68de6fcdbd6_autotoc_md254}{}\doxysubsection{\texorpdfstring{Proxies}{Proxies}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_28930e523f2ca815c97eac68de6fcdbd6_autotoc_md254}
Downstream packages should utilize the {\ttfamily initialize\+Proxy} function to add HTTP(\+S) proxy support. If the environment variable {\ttfamily ELECTRON\+\_\+\+GET\+\_\+\+USE\+\_\+\+PROXY} is set, it is called automatically.
