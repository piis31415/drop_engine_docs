JSON5 is an extension to the popular \href{https://tools.ietf.org/html/rfc7159}{\texttt{ JSON}} file format that aims to be easier to {\bfseries{write and maintain {\itshape by hand} (e.\+g. for config files)}}. It is {\itshape not intended} to be used for machine-\/to-\/machine communication. (Keep using JSON or other file formats for that. ðŸ™‚)

JSON5 was started in 2012, and as of 2022, now gets {\bfseries{\href{https://www.npmjs.com/package/json5}{\texttt{ \texorpdfstring{$>$}{>}65M downloads/week}}}}, ranks in the {\bfseries{\href{https://gist.github.com/anvaka/8e8fa57c7ee1350e3491}{\texttt{ top 0.\+1\%}}}} of the most depended-\/upon packages on npm, and has been adopted by major projects like {\bfseries{\href{https://source.chromium.org/chromium/chromium/src/+/main:third_party/blink/renderer/platform/runtime_enabled_features.json5;drc=5de823b36e68fd99009a29281b17bc3a1d6b329c}{\texttt{ Chromium}}, \href{https://github.com/vercel/next.js/blob/b88f20c90bf4659b8ad5cb2a27956005eac2c7e8/packages/next/lib/find-config.ts\#L43-L46}{\texttt{ Next.\+js}}, \href{https://babeljs.io/docs/en/config-files\#supported-file-extensions}{\texttt{ Babel}}, \href{https://community.retool.com/t/i-am-attempting-to-append-several-text-fields-to-a-google-sheet-but-receiving-a-json5-invalid-character-error/7626}{\texttt{ Retool}}, \href{https://www.jetbrains.com/help/webstorm/json.html}{\texttt{ Web\+Storm}}, and \href{https://github.com/json5/json5/wiki/In-the-Wild}{\texttt{ more}}}}. It\textquotesingle{}s also natively supported on {\bfseries{\href{https://developer.apple.com/documentation/foundation/jsondecoder/3766916-allowsjson5}{\texttt{ Apple platforms}}}} like {\bfseries{Mac\+OS}} and {\bfseries{i\+OS}}.

Formally, the {\bfseries{\href{https://spec.json5.org/}{\texttt{ JSON5 Data Interchange Format}}}} is a superset of JSON (so valid JSON files will always be valid JSON5 files) that expands its syntax to include some productions from \mbox{[}ECMAScript 5.\+1\mbox{]} (ES5). It\textquotesingle{}s also a strict {\itshape subset} of ES5, so valid JSON5 files will always be valid ES5.

This Java\+Script library is a reference implementation for JSON5 parsing and serialization, and is directly used in many of the popular projects mentioned above (where e.\+g. extreme performance isn\textquotesingle{}t necessary), but others have created \href{https://github.com/json5/json5/wiki/In-the-Wild}{\texttt{ many other libraries}} across many other platforms.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_247c485119451972b7a416015e751dd85_autotoc_md6496}{}\doxysection{\texorpdfstring{Summary of Features}{Summary of Features}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_247c485119451972b7a416015e751dd85_autotoc_md6496}
