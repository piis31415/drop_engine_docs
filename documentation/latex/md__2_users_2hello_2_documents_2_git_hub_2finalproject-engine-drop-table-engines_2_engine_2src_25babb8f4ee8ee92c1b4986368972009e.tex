{\ttfamily \doxylink{class_napi_1_1_object_reference}{Napi\+::\+Object\+Reference}} is a subclass of \doxysectlink{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2fe3873f06559c5a0ab6a1717ffe1ef05}{{\ttfamily Napi\+::Reference}}{0}, and is equivalent to an instance of {\ttfamily \doxylink{class_napi_1_1_reference}{Napi\+::\+Reference}\texorpdfstring{$<$}{<}Object\texorpdfstring{$>$}{>}}. This means that a {\ttfamily \doxylink{class_napi_1_1_object_reference}{Napi\+::\+Object\+Reference}} holds a \doxysectlink{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_2664783051e3556aa75aa8a5d3640a968}{{\ttfamily Napi\+::Object}}{0}, and a count of the number of references to that Object. When the count is greater than 0, an Object\+Reference is not eligible for garbage collection. This ensures that the Object being held as a value of the Object\+Reference will remain accessible, even if the original Object no longer is. However, Object\+Reference is unique from a Reference since properties can be set and get to the Object itself that can be accessed through the Object\+Reference.
