The {\ttfamily queue\+Microtask} function is a WHATWG standard. It queues a microtask to be executed prior to control returning to the event loop.

A microtask is a short function which will run after the current task has completed its work and when there is no other code waiting to be run before control of the execution context is returned to the event loop.

The code {\ttfamily queue\+Microtask(fn)} is equivalent to the code {\ttfamily Promise.\+resolve().then(fn)}. It is also very similar to \href{https://nodejs.org/api/process.html\#process_process_nexttick_callback_args}{\texttt{ {\ttfamily process.\+next\+Tick(fn)}}} in Node.

Using microtasks lets code run without interfering with any other, potentially higher priority, code that is pending, but before the JS engine regains control over the execution context.

See the \href{https://html.spec.whatwg.org/multipage/timers-and-user-prompts.html\#microtask-queuing}{\texttt{ spec}} or \href{https://nodejs.org/api/globals.html\#globals_queuemicrotask_callback}{\texttt{ Node documentation}} for more information.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_24eeacbc8acb0094e4f7f37b9f97dc0e4_autotoc_md8968}{}\doxysection{\texorpdfstring{Who is this package for?}{Who is this package for?}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_24eeacbc8acb0094e4f7f37b9f97dc0e4_autotoc_md8968}
This package allows you to use {\ttfamily queue\+Microtask} safely in all modern JS engines. Use it if you prioritize small JS bundle size over support for old browsers.

If you just need to support Node 12 and later, use {\ttfamily queue\+Microtask} directly. If you need to support all versions of Node, use this package.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_24eeacbc8acb0094e4f7f37b9f97dc0e4_autotoc_md8969}{}\doxysection{\texorpdfstring{Why not use {\ttfamily process.\+next\+Tick}?}{Why not use {\ttfamily process.\+next\+Tick}?}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_24eeacbc8acb0094e4f7f37b9f97dc0e4_autotoc_md8969}
In Node, {\ttfamily queue\+Microtask} and {\ttfamily process.\+next\+Tick} are \href{https://nodejs.org/api/globals.html\#globals_queuemicrotask_callback}{\texttt{ essentially equivalent}}, though there are \href{https://github.com/YuzuJS/setImmediate\#macrotasks-and-microtasks}{\texttt{ subtle differences}} that don\textquotesingle{}t matter in most situations.

You can think of {\ttfamily queue\+Microtask} as a standardized version of {\ttfamily process.\+next\+Tick} that works in the browser. No need to rely on your browser bundler to shim {\ttfamily process} for the browser environment.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_24eeacbc8acb0094e4f7f37b9f97dc0e4_autotoc_md8970}{}\doxysection{\texorpdfstring{Why not use {\ttfamily set\+Timeout(fn, 0)}?}{Why not use {\ttfamily set\+Timeout(fn, 0)}?}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_24eeacbc8acb0094e4f7f37b9f97dc0e4_autotoc_md8970}
This approach is the most compatible, but it has problems. Modern browsers throttle timers severely, so {\ttfamily set\+Timeout(…, 0)} usually takes at least 4ms to run. Furthermore, the throttling gets even worse if the page is backgrounded. If you have many {\ttfamily set\+Timeout} calls, then this can severely limit the performance of your program.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_24eeacbc8acb0094e4f7f37b9f97dc0e4_autotoc_md8971}{}\doxysection{\texorpdfstring{Why not use a microtask library like \href{https://www.npmjs.com/package/immediate}{\texttt{ {\ttfamily immediate}}} or \href{https://www.npmjs.com/package/asap}{\texttt{ {\ttfamily asap}}}?}{Why not use a microtask library like \href{https://www.npmjs.com/package/immediate}{\texttt{ {\ttfamily immediate}}} or \href{https://www.npmjs.com/package/asap}{\texttt{ {\ttfamily asap}}}?}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_24eeacbc8acb0094e4f7f37b9f97dc0e4_autotoc_md8971}
These packages are great! However, if you prioritize small JS bundle size over optimal performance in old browsers then you may want to consider this package.

This package ({\ttfamily queue-\/microtask}) is four times smaller than {\ttfamily immediate}, twice as small as {\ttfamily asap}, and twice as small as using {\ttfamily process.\+next\+Tick} and letting the browser bundler shim it automatically.

Note\+: This package throws an exception in JS environments which lack {\ttfamily Promise} support -- which are usually very old browsers and Node.\+js versions.

Since the {\ttfamily queue\+Microtask} API is supported in Node.\+js, Chrome, Firefox, Safari, Opera, and Edge, {\bfseries{the vast majority of users will get optimal performance}}. Any JS environment with {\ttfamily Promise}, which is almost all of them, also get optimal performance. If you need support for JS environments which lack {\ttfamily Promise} support, use one of the alternative packages.\hypertarget{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_24eeacbc8acb0094e4f7f37b9f97dc0e4_autotoc_md8972}{}\doxysection{\texorpdfstring{What is a shim?}{What is a shim?}}\label{md__2_users_2hello_2_documents_2_git_hub_2finalproject-engine-drop-table-engines_2_engine_2src_24eeacbc8acb0094e4f7f37b9f97dc0e4_autotoc_md8972}
