C++ code needs to be compiled into executable form whether it be as an object file to linked with others, a shared library, or a standalone executable.

The main reason for this is that we need to link to the Node.\+js dependencies and headers correctly, another reason is that we need a cross platform way to build C++ source into binary for the target platform.

Until now {\bfseries{node-\/gyp}} is the {\bfseries{de-\/facto}} standard build tool for writing Node.\+js addons. It\textquotesingle{}s based on Google\textquotesingle{}s {\bfseries{gyp}} build tool, which abstract away many of the tedious issues related to cross platform building.

{\bfseries{node-\/gyp}} uses a file called {\ttfamily binding.\+gyp} that is located on the root of your addon project.
